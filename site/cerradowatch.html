// fontes de dados · funcionalidades

Quatro domínios, um pipeline

Cada fonte tem seu conector Python com validação de schema na borda, upsert idempotente e testes de qualidade no dbt.

🔥
Queimadas — NASA FIRMS
Satélite VIIRS/SNPP com resolução espacial de 375m e latência de ~3h. Dados ingeridos semanalmente com upsert idempotente — sem duplicatas mesmo que o pipeline rode duas vezes.
NASA FIRMS API · VIIRS · resolução 375m
🌳
Desmatamento — INPE PRODES
Série histórica de desmatamento desde 2010, por estado, via TerraBrasilis WFS. Modelado no dbt mart com crescimento acumulado e ranking por UF para análise comparativa.
INPE PRODES · TerraBrasilis WFS · desde 2010
🌡️
Clima — INMET
Temperatura, precipitação e umidade de estações automáticas do Cerrado. Cálculo de índice de risco de seca no dbt mart — correlação direta com picos de queimadas.
INMET · estações automáticas · dados mensais
🌾
Preços Agrícolas — CONAB
Soja, milho e algodão por estado — commodities que dirigem a expansão do agronegócio sobre o Cerrado. O mart cruza preços com desmatamento para revelar correlações.
CONAB · mensal · soja · milho · algodão
⚙️
dbt — Transformação em 2 Camadas
Staging limpa, tipifica e padroniza os dados brutos. Mart constrói as tabelas analíticas finais com testes de qualidade automáticos — fonte única da verdade para o dashboard.
dbt-core · staging → mart · testes de qualidade
🤖
Orquestração — GitHub Actions
Cron semanal toda segunda às 06h BRT executa o pipeline completo: ingestão + dbt run + validação de qualidade + alerta de queimadas por e-mail. Zero infra, auditável, sem custo operacional.
GitHub Actions cron · segunda 06h BRT · alertas e-mail

"Construí um pipeline de dados ambiental de produção que ingere dados reais de NASA FIRMS, INPE PRODES, INMET e CONAB, transforma com dbt em duas camadas (staging → mart) e serve um dashboard interativo ao vivo via Streamlit. O pipeline roda automaticamente toda semana via GitHub Actions cron, com validação de qualidade de dados e alertas de queimadas por e-mail."
